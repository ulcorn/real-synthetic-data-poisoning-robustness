# Data Poisoning Robustness: Real vs Synthetic Tabular Data

Экспериментальное исследование устойчивости ML-моделей к атакам отравления данных в зависимости от источника обучающих данных (реальные, синтетические, смешанные).

**Автор:** Корнилова Ульяна, НИУ ВШЭ, ФКН, 2026  
**Научный руководитель:** Силаев Юрий Владимирович

## Эксперименты

### Параметры

| Параметр | Значение |
|---|---|
| Датасеты | Adult Census Income (32 561), Bank Marketing (41 188) |
| Модели | LogisticRegression, XGBoost, MLP (только BD) |
| Генераторы | CTGAN, TVAE (SDV, 300 эпох) |
| Атаки | LF (label flipping), FP (feature perturbation), BD (backdoor) |
| Доля отравления $\rho$ | 0.5%, 1%, 2%, 5%, 10% |
| Доля реальных данных $\alpha$ | $\{0.0, 0.25, 0.5, 0.75, 1.0\}$ |
| Сила FP $\lambda$ | $\{0.5, 1.0, 2.0\}$ |
| Триггер BD | $z = 1.5$ на двух числовых признаках |
| Инициализации | $\{42, 123, 456, 789, 1024\}$ |
| Метрика качества | ROC-AUC на реальной отложенной выборке |
| Метрика бэкдоров | $\text{ASR}_{\text{nontarget}}$ |

### Режимы отравления

- **R1:** отравление после смешивания - сначала формируется $D(\alpha)$, затем доля $\rho$ точек отравляется
- **R2:** синтетическая инъекция - генератор сэмплирует отравленные точки, которые замещают долю $\rho$ в $D(\alpha)$

### Защиты

- **Санитизация:** удаление выбросов (EllipticEnvelope, contamination=5%)
- **Спектральная фильтрация:** SVD на представлениях MLP, удаление 5% точек с максимальной проекцией
- **Cleanlab:** очистка меток через cross-validated confidence (направление дальнейшей работы)

## Метрики

### Качество модели

- **ROC-AUC** на чистой реальной отложенной выборке $D_{\text{test}}$ - основная метрика. Дополнительно: accuracy и F1-score.
- **Кривые деградации:** $\Delta(\rho) = \text{AUC}_{\text{clean}} - \text{AUC}_{\text{poisoned}}$, усреднённые по 5 инициализациям ($\text{mean} \pm \text{std}$).
- **TSTR (Train-on-Synthetic, Test-on-Real):** разрыв AUC между обучением на реальных и полностью синтетических данных. Adult: 0.013, Bank: 0.020 - синтетика пригодна для downstream-обучения.

### Метрика бэкдоров

- **ASR nontarget:** $P(f(t(x)) = y^\dagger \mid y(x) \neq y^\dagger)$ — доля успешных атак только на объектах с истинной меткой, отличной от целевой. Исключает влияние дисбаланса классов и корреляции триггера с целевой меткой.
- $\text{ASR} = P(f(t(x)) = y^\dagger \mid x \in D_{\text{test}})$ - полная ASR, сохраняется для совместимости с предыдущими работами.
- $\text{ASR}_{\Delta} = \text{ASR}(\rho) - \text{ASR}(\rho{=}0)$ - baseline-adjusted: чистый эффект атаки с поправкой на естественный отклик модели на триггерный паттерн.

### Метрики приватности

- **MIA advantage** $= |P(\hat{m}{=}1 \mid m{=}1) - P(\hat{m}{=}1 \mid m{=}0)|$ - превышение точности атаки вывода принадлежности над случайным угадыванием. Значения $< 0.01$ считаются безопасными.
- **DCR** (Distance-to-Closest-Record) $= \min_{r \in D_{\text{train}}} d(s, r)$ - минимальное евклидово расстояние от синтетической записи до ближайшей реальной. Низкие значения указывают на копирование записей. Медиана: CTGAN 1.62, TVAE 1.09–1.16.
- **NNDR** (Nearest-Neighbor Distance Ratio) $= d(s, r_1) / d(s, r_2)$ - отношение расстояний до первого и второго ближайших реальных соседей. Значения $\approx 1.0$ указывают на отсутствие массовой меморизации; значения $\ll 1.0$ - на кластеризацию синтетических точек вокруг отдельных реальных записей.

## Основные результаты

### H1: Источник данных влияет на устойчивость (частично подтверждается)

- XGBoost на синтетике устойчивее к LF на 30% ($\Delta = 0.008$ vs $0.012$)
- Зависимость от $\alpha$ нелинейна
- В R2 деградация LogReg на реальных данных в $10\times$ выше (Bank: $0.018$ vs $0.002$)
- Для BD: LogReg и MLP уязвимее на синтетике ($\text{ASR}_{\text{nontarget}}$ на 5–10% выше), XGBoost нечувствителен к $\alpha$

### H2: Выбор генератора существенно влияет (подтверждается)

- TVAE: деградация XGBoost при LF в $4\times$ сильнее ($\Delta = 0.032$ vs $0.008$)
- BD: TVAE MLP при $\rho = 5\%$ достигает $\text{ASR}_{\text{nontarget}} = 0.99$ (CTGAN: $0.90$)
- Privacy: медиана DCR для TVAE $1.16$ vs $1.62$ у CTGAN

### H3: Защиты зависят от источника данных (подтверждается)

- Санитизация деструктивна на TVAE ($\Delta = -0.064$) и на реальных данных в R2 ($\Delta = -0.038$)
- Спектральная фильтрация: $|\Delta|$ до $0.03$– $0.06$, но направление нестабильно
- XGBoost критически уязвим к BD: $\text{ASR}_{\text{nontarget}} = 0.99$ при $\rho = 0.5\%$, Clean AUC неизменен
